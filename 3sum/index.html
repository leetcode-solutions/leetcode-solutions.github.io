3Sum

Medium

https://leetcode.com/problems/3sum/

Problem#

Given an integer array nums, return all the triplets [nums[i], nums[j], nums[k]] such that i != j, i != k, and j != k, and nums[i] + nums[j] + nums[k] == 0.

Notice that the solution set must not contain duplicate triplets.

Examples#

Inputnums = [-1,0,1,2,-1,-4]
Output[[-1,-1,2],[-1,0,1]]

Constraints#

  • 0 <= nums.length <= 3000
  • -105 <= nums[i] <= 105

Hints#

Solution#

A naive approach may be to test all three element combinations of the array; however, there are O(n3)O(n^3)O(n3) possible combinations, which is far too slow.

Suppose we fix some element xxx in numsnumsnums. We can observe then that we are then solving a generalization of Two Sum, where we want to find two other elements that sum to −x-x−x. Knowing that we can solve for Two Sum in O(n)O(n)O(n) time and space, we can thus fix each element of numsnumsnums and solve for Two Sum accordingly, resulting in overall O(n2)O(n^2)O(n2) time and O(n)O(n)O(n) space (assuming the space used for Two Sum is reused between iterations).

While we've greatly optimized relative to the brute force approach, we still test up to all n−1n-1n−1 other elements when solving for Two Sum. We want to see if we can reduce the search space for solving Two Sum.

Suppose we preprocess numsnumsnums by sorting it in ascending order. While an O(nlog⁡n)O(n\log n)O(nlogn) operation, it is still less than our O(n2)O(n^2)O(n2) solution so it likely doesn't have a too large impact on runtime. From the resultant numssortednums_{sorted}numssorted​, for each index iii, we can then similarly fix x=nums[i]x=nums[i]x=nums[i] and solve for Two Sum, with the subarray nums[i+1:]nums[i+1:]nums[i+1:] as the search space. This brings the overall runtime to O(nlog⁡n)+O(1+…+n−1)=O(nlog⁡n)+O(n22)O(n\log n) + O(1 + \ldots + n-1) = O(n\log n) + O(\frac{n^2}{2})O(nlogn)+O(1+…+n−1)=O(nlogn)+O(2n2​) - asymptotically still the same, but in practice for large nnn, potentially saving time with the lower coefficient on the dominant term.

As a side effect of sorting the array, it also becomes easier to avoid duplicate values: since the elements are sorted, whenever we want to move on to the "next element", we can easily check if it's duplicate by comparing to the previous value, and continue to move until the new value is different from the previous.

We can also observe that since we're solving Two Sum for −x-x−x and that we're traversing the array in ascending order, if xxx is ever greater than zero, then there are no more possible solutions, since the target is negative and the pool of candidates is positive.

We can further observe that to solve Two Sum when the array is sorted, we can use two pointers instead of the standard hash table solution: beginning with indeces j=i+1,k=n−1j = i+1, k = n-1j=i+1,k=n−1, we can shift jjj and kkk inward based on their sum relative to the target −x-x−x. This Two Sum algorithm identifies whether there is a solution in a single pass while using O(1)O(1)O(1) space, a marked improvement over the hash table implementation. As a result, our overall algorithm, while still O(n2)O(n^2)O(n2) time complexity, reduces to O(1)O(1)O(1) space complexity (excluding return values, and assuming the input array numsnumsnums is mutable; otherwise, space complexity is still O(n)O(n)O(n)).

class Solution:    def threeSum(self, nums: List[int]) -> List[List[int]]:        n = len(nums)        nums.sort()                def next_uniq(i, bound):            i += 1            while i < bound and nums[i] == nums[i-1]:                i += 1            return i        def prev_uniq(i, bound):            i -= 1            while i > bound and nums[i] == nums[i+1]:                i -= 1            return i        res = []        for i in range(n - 2):            if nums[i] > 0:                break                        if i > 0 and nums[i] == nums[i-1]:                continue            j, k = i + 1, n - 1            while j < k:                s = nums[i] + nums[j] + nums[k]                if s == 0:                    res.append([nums[i], nums[j], nums[k]])                    j = next_uniq(j, k)                    k = prev_uniq(k, j)                elif s < 0:                    j = next_uniq(j, k)                else:                    k = prev_uniq(k, j)        return res
